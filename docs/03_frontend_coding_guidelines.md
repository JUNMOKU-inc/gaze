```

├── components/

│   │   └── types.ts

│   ├── Preview/               # Post-capture quick preview
│   ├── Settings/              # Preferences UI
│   └── common/                # Button, Tooltip, IconButton, etc.
├── hooks/                     # App-wide custom hooks

├── styles/
│   └── index.css              # Tailwind entry + CSS custom properties
└── main.tsx                   # React root + Tauri event bootstrap
```

Key rules:
- No `react-router`. This is a desktop app with discrete windows managed by Tauri, not a web SPA. Use a discriminated-union view state instead.
- One component per file. Co-locate hooks, types, and sub-components in the same directory.
- `lib/bindings.ts` is generated by `tauri-specta` -- never hand-edit it.

## 2. Component Patterns

### View switching (no router)

```tsx
// stores/appStore.ts
type View =
  | { kind: "idle" }
  | { kind: "preview"; imageData: string }
  | { kind: "editor"; imageData: string; captureId: string }
  | { kind: "settings" };

interface AppState {
  view: View;
  setView: (v: View) => void;
}

export const useAppStore = create<AppState>((set) => ({
  view: { kind: "idle" },
  setView: (view) => set({ view }),
}));
```

```tsx
// App.tsx
function App() {
  const view = useAppStore((s) => s.view);

  switch (view.kind) {
    case "idle":
      return null; // tray-only, no visible window
    case "preview":
      return <Preview imageData={view.imageData} />;
    case "editor":
      return <AnnotationEditor imageData={view.imageData} captureId={view.captureId} />;
    case "settings":
      return <Settings />;
  }
}
```

### Component conventions

- Props interfaces live alongside the component, named `<Component>Props`.
- Prefer `function` declarations over arrow functions for top-level components.
- Keep components under ~150 lines. Extract hooks when logic grows.

```tsx
interface PreviewProps {
  imageData: string;
}

function Preview({ imageData }: PreviewProps) {
  // ...
}
```

## 3. State Management (Zustand)

Use **Zustand** for all shared state. Keep stores small and domain-scoped.

```tsx
// stores/captureStore.ts
import { create } from "zustand";

interface CaptureState {
  lastCapture: CaptureResult | null;
  isCapturing: boolean;
  startCapture: () => void;
  setCaptureResult: (result: CaptureResult) => void;
}

export const useCaptureStore = create<CaptureState>((set) => ({
  lastCapture: null,
  isCapturing: false,
  startCapture: () => set({ isCapturing: true }),
  setCaptureResult: (result) => set({ lastCapture: result, isCapturing: false }),
}));
```

Rules:
- **No Context providers for global state.** Zustand stores are module-level singletons.
- **Select only what you need** to avoid unnecessary re-renders: `useStore((s) => s.field)`.
- **Async actions** (Tauri invocations) live inside store actions, not in components.
- Use `subscribeWithSelector` middleware if you need fine-grained subscriptions.

## 4. Tauri Bridge (invoke pattern)

### Recommended: tauri-specta (auto-generated bindings)

On the Rust side, `tauri-specta` generates `src/lib/bindings.ts` with fully typed command wrappers.

```rust
// src-tauri/src/main.rs
#[tauri::command]
#[specta::specta]
fn capture_area(x: i32, y: i32, w: u32, h: u32) -> Result<CaptureResult, String> {
    // ...
}

fn main() {
    let builder = tauri_specta::ts::builder()
        .commands(tauri_specta::collect_commands![capture_area, optimize_for_llm])
        .events(tauri_specta::collect_events![CaptureComplete]);

    #[cfg(debug_assertions)]
    builder.export(specta_typescript::Typescript::default(), "../src/lib/bindings.ts")
        .expect("Failed to export TS bindings");

    tauri::Builder::default()
        .invoke_handler(builder.invoke_handler())
        .setup(move |app| { builder.mount_events(app); Ok(()) })
        .run(tauri::generate_context!())
        .expect("error running app");
}
```

Frontend usage -- fully type-safe, no manual `invoke()`:

```tsx
import { commands } from "../lib/bindings";

async function handleCapture(region: Region) {
  const result = await commands.captureArea(region.x, region.y, region.w, region.h);
  // `result` is typed as CaptureResult -- no `as` cast needed
  useCaptureStore.getState().setCaptureResult(result);
}
```

### Tauri event listener hook

```tsx
// hooks/useTauriEvent.ts
import { listen, type UnlistenFn } from "@tauri-apps/api/event";
import { useEffect } from "react";

export function useTauriEvent<T>(event: string, handler: (payload: T) => void) {
  useEffect(() => {
    let unlisten: UnlistenFn | undefined;
    listen<T>(event, (e) => handler(e.payload)).then((fn) => {
      unlisten = fn;
    });
    return () => { unlisten?.(); };
  }, [event, handler]);
}
```

Rules:
- **Never call raw `invoke()` directly.** Always go through the generated `commands` object.
- **All Rust commands return `Result<T, String>`.** Handle errors explicitly on the TS side.
- **Binary data** (images): pass as `Vec<u8>` from Rust; arrives as `number[]` in TS. Convert with `new Uint8Array(data)`.

## 5. TypeScript Strictness Rules

### tsconfig.json

```jsonc
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "exactOptionalPropertyTypes": true,
    "forceConsistentCasingInFileNames": true,
    "verbatimModuleSyntax": true
  }
}
```

### Code-level rules

| Rule | Rationale |
|------|-----------|
| **No `any`** | Use `unknown` and narrow with type guards. ESLint `@typescript-eslint/no-explicit-any`. |
| **No `as` casts** | Prefer type guards or `satisfies`. Exception: `as const` is fine. |
| **Discriminated unions for state** | All state variants have a `kind` literal field (see View type above). |
| **Exhaustive switches** | Helper: `function assertNever(x: never): never { throw new Error("Unexpected: " + x); }` |
| **Readonly by default** | Prefer `readonly` arrays and `Readonly<T>` for props. |

## 6. Canvas / Annotation Editor (Konva.js)

### Architecture

```
AnnotationEditor/
├── AnnotationEditor.tsx       # Stage + Layer setup, tool dispatch
├── types.ts                   # AnnotationObject union type
├── hooks/
│   ├── useCanvasTool.ts       # Current tool state machine
│   └── useAnnotationHistory.ts # Undo/redo stack
└── tools/
    ├── ArrowTool.tsx           # Renders + handles arrow creation
    ├── TextTool.tsx
    ├── BlurTool.tsx
    └── HighlightTool.tsx
```

### Core pattern: State in React, rendering in Konva

```tsx
// Keep annotation objects in React state, render declaratively
type AnnotationObject =
  | { kind: "arrow"; id: string; points: number[]; color: string }
  | { kind: "text"; id: string; x: number; y: number; text: string }
  | { kind: "blur"; id: string; x: number; y: number; w: number; h: number };

function AnnotationEditor({ imageData }: { imageData: string }) {
  const [objects, setObjects] = useState<readonly AnnotationObject[]>([]);

  return (
    <Stage width={width} height={height}>
      {/* Static layer: background image (cached for performance) */}
      <Layer listening={false}>
        <Image image={bgImage} />
      </Layer>
      {/* Dynamic layer: annotations */}
      <Layer>
        {objects.map((obj) => (
          <AnnotationRenderer key={obj.id} object={obj} />
        ))}
      </Layer>
    </Stage>
  );
}
```

### Performance rules

| Technique | Why |
|-----------|-----|
| **Separate layers** for static (background image) and dynamic (annotations) content | Each `Layer` is a separate `<canvas>`. Static layer never redraws. |
| **`listening={false}`** on background layer and non-interactive shapes | Eliminates hit-detection overhead. |
| **`shape.cache()`** for complex shapes (blur regions, grouped annotations) | Renders to offscreen bitmap; redraws are instant. |
| **`perfectDrawEnabled={false}`** on shapes with fill + stroke | Skips double-render pass. |
| **`batchDraw()`** when updating multiple shapes at once | Coalesces into a single render frame. |
| Selection state stored as **`Set<string>` of IDs** in React, not on Konva nodes | Single source of truth; avoids sync issues. |

## 7. Testing Strategy

### Setup: Vitest + React Testing Library + Tauri mocks

```ts
// vitest.config.ts
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    environment: "jsdom",
    globals: true,
    setupFiles: ["./src/test/setup.ts"],
    css: true,
  },
});
```

```ts
// src/test/setup.ts
import { cleanup } from "@testing-library/react";
import { afterEach, beforeAll } from "vitest";
import { randomFillSync } from "crypto";

// Tauri requires crypto in test environment
beforeAll(() => {
  Object.defineProperty(window, "crypto", {
    value: { getRandomValues: (buf: Buffer) => randomFillSync(buf) },
  });
});

afterEach(() => {
  cleanup();
});
```

### Mocking Tauri commands

```ts
import { mockIPC, clearMocks } from "@tauri-apps/api/mocks";
import { afterEach, test, expect, vi } from "vitest";
import { render, screen } from "@testing-library/react";
import userEvent from "@testing-library/user-event";

afterEach(() => clearMocks());

test("Settings saves preferences via Tauri command", async () => {
  const saveSpy = vi.fn();
  mockIPC((cmd, args) => {
    if (cmd === "save_settings") {
      saveSpy(args);
      return { ok: true };
    }
  });

  render(<Settings />);
  await userEvent.click(screen.getByRole("button", { name: /save/i }));

  expect(saveSpy).toHaveBeenCalledWith(
    expect.objectContaining({ theme: "system" })
  );
});
```

### What to test

| Test | Priority |
|------|----------|
| Store logic (Zustand actions, state transitions) | **High** -- pure logic, fast, no DOM |
| Components that invoke Tauri commands (with mockIPC) | **High** -- verifies the bridge contract |
| Discriminated union exhaustiveness (compile-time) | **High** -- TypeScript compiler handles this |
| Annotation tool state machines | **Medium** -- test via hook unit tests |
| Visual canvas output | **Skip** -- use manual QA or screenshot tests |
| Tauri window management / OS integration | **Skip** -- test in E2E or manual |

### Test file naming

Place tests next to source: `Preview.tsx` -> `Preview.test.tsx`.

## 8. Styling Approach

### Tailwind CSS v4

- Utility-first for all layout, spacing, color. No CSS modules.
- Use `@theme` in `src/styles/index.css` for Gaze design tokens.

```css
/* src/styles/index.css */
@import "tailwindcss";

@theme {
  --color-surface: oklch(0.15 0.01 260);
  --color-surface-hover: oklch(0.20 0.01 260);
  --color-accent: oklch(0.65 0.25 250);
  --radius-panel: 10px;
}
```

### Dark mode / system theme

Tauri apps should respect the OS theme. Use Tailwind's `dark:` variant with `media` strategy (default in v4).

```tsx
// hooks/useSystemTheme.ts
import { useEffect, useState } from "react";

export function useSystemTheme() {
  const [isDark, setIsDark] = useState(
    () => window.matchMedia("(prefers-color-scheme: dark)").matches
  );

  useEffect(() => {
    const mq = window.matchMedia("(prefers-color-scheme: dark)");
    const handler = (e: MediaQueryListEvent) => setIsDark(e.matches);
    mq.addEventListener("change", handler);
    return () => mq.removeEventListener("change", handler);
  }, []);

  return isDark ? "dark" : "light";
}
```

### Desktop-specific styling rules

- **No hover states that hide critical actions.** Desktop users expect visible controls.
- **Use `select-none` on all non-text UI** to prevent accidental text selection during drag.
- **Frameless window**: add `-webkit-app-region: drag` on the title bar div for window dragging.
- **Minimum touch target**: 28px (desktop), not 44px (mobile).
- **WebKit/WebView2 parity**: avoid bleeding-edge CSS. Test both macOS (WebKit) and Windows (Chromium).

## 9. Code Review Checklist

Before merging any frontend PR, verify:

### Correctness
- [ ] No `any` types or `as` casts (except `as const`)
- [ ] Tauri commands called via generated bindings, not raw `invoke()`
- [ ] Error results from Rust commands are handled (try/catch or `.catch()`)
- [ ] Zustand selectors select minimal state (`(s) => s.field`, not `(s) => s`)
- [ ] `useEffect` cleanup: all listeners (`listen()`, event listeners) are cleaned up

### Performance
- [ ] Canvas: static content on a separate `Layer` with `listening={false}`
- [ ] No inline object/array literals in props that would cause re-renders
- [ ] Large lists use `React.memo` or are virtualized

### Testing
- [ ] New Tauri command integrations have a `mockIPC` test
- [ ] Store actions have unit tests
- [ ] No snapshot tests (they bitrot fast; prefer explicit assertions)

### Styling
- [ ] Works in both light and dark mode
- [ ] No hardcoded colors outside of `@theme` tokens
- [ ] Draggable regions use `select-none` and `-webkit-app-region: drag`

### Desktop
- [ ] Keyboard shortcuts don't conflict with OS defaults
- [ ] Window state (size, position) is not assumed -- works at any dimension
- [ ] No `window.location` navigation -- state changes go through Zustand
