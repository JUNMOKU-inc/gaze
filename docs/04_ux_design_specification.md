# Gaze — UX設計仕様書

> 作成日: 2026-03-25
> ステータス: v1 UX設計

---

## 設計原則

1. **ゼロフリクション**: ハッピーパスは「1ホットキー + 1ペースト」。クリック不要。
2. **プログレッシブ・ディスクロージャー**: パワー機能は存在するが、高速パスを阻害しない。
3. **周辺的気づき**: フィードバックは見えるが、注意を強制しない。
4. **LLMネイティブ**: トークン数と最適化が全UIに一貫して存在する。
5. **macOSネイティブ**: SF Symbols、システムフォント、Vibrancy、標準コントロール。

## 1. オンボーディング（初回起動）

### 画面1: パーミッション取得（唯一の必須画面）

**レイアウト**: 中央配置、フレームレスウィンドウ（480x560px）、ダーク背景

**コンテンツ**:
- Gazeアイコン（64px）+ ワードマーク
- ワンライナー: "Capture. Optimize. Paste into any LLM."
- アニメーションデモ（4秒ループ）: ホットキー → エリア選択 → プレビュー → Claude にペースト
- CTAボタン: **"Enable Screen Capture"**

**CTAクリック時**:
1. macOS権限ダイアログをトリガー
2. 500ms間隔でポーリング（権限付与を待機）
3. 付与されたら画面2にクロスフェード（200ms）
4. 拒否された場合: System Settings への誘導ボタン + "Check Again" ボタン

### 画面2: ショートカット + プロバイダ設定

**左列 — ショートカット（事前設定済み、変更可能）**:

| アクション | デフォルト | アイコン |
|-----------|----------|---------|
| エリアキャプチャ | `⌘ ⇧ 2` | 点線矩形 |
| ウィンドウキャプチャ | `⌘ ⇧ 1` | ウィンドウ |
| フルスクリーン | `⌘ ⇧ 3` | モニター |
| OCRキャプチャ | `⌘ ⇧ 4` | テキストスキャン |

ショートカットの競合を自動検出。macOS標準スクショとの競合は警告表示。

**右列 — デフォルトLLMプロバイダ**:

3枚のカード（Claude / GPT-4o / Gemini）。各カードに最適化サマリー表示。**Claude がプリセレクト**。

**CTA**: "Start Capturing" → ウィンドウ消失、メニューバーアイコン出現、通知表示。

**合計オンボーディング時間**: 45秒以内。画面は2つのみ。

## 2. エリアキャプチャ（最重要フロー）

### Phase 1: 起動（0ms – 80ms）

**トリガー**: `⌘ ⇧ 2`

1. **ホットキー検出と同時に**画面全体のビットマップを取得（オーバーレイ表示前にキャプチャ → オーバーレイがキャプチャに映り込まない）
2. フルスクリーン透明オーバーレイウィンドウを作成
3. 画面が40%暗転（`rgba(0,0,0,0.6)` セミ透明黒）
4. カーソルが精密クロスヘア（`+`）に変化
5. カーソル近くにHUD表示: `0 × 0`（ピル型バッジ、モノスペース11px）

### Phase 2: 選択（300ms – 2000ms）

**操作**: クリック＆ドラッグ（シングルインタラクション）

**ドラッグ中**:
- 選択範囲は暗転を切り抜き（フル輝度表示）→ 選択/非選択の明確なコントラスト
- 1px破線ボーダー（白、アニメーション）で選択範囲を囲む
- ディメンションHUD: `1284 × 768` + 推定トークン数 `~1,312 tok`（リアルタイム更新）

**精密操作**:
- `⇧` ホールド: 正方形に制約
- `⌥` ホールド: 中心から拡張
- マウスリリース後: 8つのハンドル表示（コーナー + 中点）で微調整可能
- 矢印キー: 1px nudge（`⇧` + 矢印で10px）
- `Enter` / `Space` で確定、`Esc` でキャンセル
- エッジ近くでルーペ表示（5倍、120x120px円形）

### Phase 3: キャプチャ確定（0ms – 150ms）

1. **視覚フィードバック（0-50ms）**: 選択範囲が白くフラッシュ + 軽いシャッター音
2. **画像処理（20-80ms）**:
   - ビットマップをクロップ
   - LLM最適化パイプライン実行（リサイズ → フォーマット変換）
   - クリップボードに書き込み（`arboard`経由）
3. **オーバーレイ消失（50-150ms）**: フェードアウト
4. **プレビューポップアップ表示**: 右下からスライドイン

### ハッピーパス・タイムライン

| 時間 | イベント |
|------|---------|
| T+0ms | `⌘⇧2` 押下 |
| T+80ms | オーバーレイ表示、クロスヘアアクティブ |
| T+80ms–T+1500ms | エリア選択（ユーザー操作） |
| T+1500ms | マウスリリース |
| T+1550ms | フラッシュ、画像処理完了、クリップボード書き込み |
| T+1650ms | オーバーレイ消失、プレビューポップアップ表示 |
| T+1700ms | Claude/ChatGPTで `⌘V` |

**ホットキーからペーストまで約1.7秒。** クリップボードへの書き込みは T+1550ms で完了 → プレビュー表示前にペースト可能。

## 3. ウィンドウキャプチャ

**トリガー**: `⌘ ⇧ 1`

### 選択

- ホバーでウィンドウをハイライト（暗転切り抜き + 2pxブルーボーダー `#4A9EFF`）
- ウィンドウ上部にHUD: `App Name — 1440 × 900 — ~1,580 tok`
- `Tab` キー: カーソル位置で重なったウィンドウを切り替え
- `⌥` ホールド: ドロップシャドウ + タイトルバーも含む
- クリックで確定 → Phase 3（エリアキャプチャと同一）

### デフォルト動作

- `⌥` なし: コンテンツ部分のみ（シャドウなし、角丸なし）→ LLM入力にはクリーン
- `⌥` あり: シャドウ + タイトルバー → ドキュメント/プレゼン向け

## 4. フルスクリーンキャプチャ

**トリガー**: `⌘ ⇧ 3`

- 即座にキャプチャ。オーバーレイなし、選択なし。
- 画面全体が白くフラッシュ（80ms）
- 最適化パイプライン実行 → クリップボード → プレビューポップアップ
- マルチモニター: カーソルがあるスクリーンのみ。`⌥` + `⌘⇧3` で全画面ステッチ。
- **ホットキーからクリップボードまで200ms以内。** 最速パス。

## 5. プレビューポップアップ（最重要UI要素）

### 外観

- **位置**: デフォルトは画面右下、端から16pxマージン
  - 設定で5つのオプションから選択可能: bottom-right（デフォルト） / bottom-left / top-right / top-left / near cursor（カーソルから20pxオフセット、画面端にクランプ）
  - 選択した位置はセッション間で永続化
- **サイズ**: 幅320px、高さ動的（220–280px）
- **スタイル**: 角丸12px、`rgba(30,30,30,0.95)` + Vibrancyブラー、ドロップシャドウ

### レイアウト

```
┌─────────────────────────────────┐
│  ┌───────────────────────────┐  │
│  │                           │  │
│  │    キャプチャサムネイル      │  │
│  │    (max 180px tall)       │  │
│  │                           │  │
│  └───────────────────────────┘  │
│                                 │
│  1284 × 768 → 1568 × 940       │  ← オリジナル → 最適化後
│  142 KB · ~1,312 tok · ☁ Claude │  ← サイズ, トークン, プロバイダ
│                                 │
│  ✓ Copied                       │  ← 確認（緑）
└─────────────────────────────────┘

  ※ サムネイルにホバーすると:
┌─────────────────────────────────┐
│  ┌───────────────────────────┐  │
│  │ ██████████████████████████ │  │
│  │ ██ セミ透明ダークオーバーレイ█│  │
│  │ ██ [Copy] [Edit] [OCR]  ██│  │
│  │ ██ [Share] [Save] [×]   ██│  │
│  │ ██████████████████████████ │  │
│  └───────────────────────────┘  │
│  ...                            │
└─────────────────────────────────┘
```

### 情報表示

- **行1**: `{元の幅} × {元の高さ} → {最適化後の幅} × {最適化後の高さ}`（最適化が行われたことを視覚的に表現）
- **行2**: `{ファイルサイズ} · ~{トークン数} tok · {プロバイダアイコン} {プロバイダ名}`
  - プロバイダ名はクリッカブル → クイックスイッチドロップダウン
- **確認バッジ**: "✓ Copied"（緑、1秒後に50%不透明度にフェード）

### アクションボタン（ホバーリビール）

**通常状態**: サムネイル + 情報行のみ表示（コンパクト）

**サムネイルにホバー時**: サムネイル上にセミ透明ダークオーバーレイ（`rgba(0,0,0,0.6)`）が表示され、以下のアクションボタンが出現:

| ボタン | ショートカット（ポップアップ表示中） | アクション |
|--------|-----|--------|
| Copy | `C` | 現在状態の画像を再コピー。注釈がある場合は注釈入り画像を優先 |
| Annotate | `E` | Expand + Focus Marker 編集モードを開く |
| OCR | `T` | OCR実行、テキストをクリップボードに |
| Share | — | 共有（将来実装） |
| Save | `S` | 現在状態の画像を `~/Desktop/Gaze_{timestamp}.png` に保存 |
| × (Dismiss) | `Esc` | 閉じる（クリップボード内容は維持） |

**ヒント表示**: 初回3回の使用時、サムネイル下部に薄い "hover for actions" ヒントを表示。3回後は非表示。

**キーボードショートカット**: `E`, `T`, `S`, `Esc` はホバーなしでもポップアップ表示中に常に動作する。

### タイミングとオートディスミス

- **デフォルト**: 5秒後に自動消失（300msフェードアウト）
- **ホバー中**: タイマー停止
- **プログレスバー**: ポップアップ最下部に2pxの薄いバー（左→右に縮小）
- **設定可能**: 3s / 5s（デフォルト） / 10s / Until action（自動ディスミスなし、ユーザー操作まで表示し続ける）

### スタッキング（連続キャプチャ）

1. 既存ポップアップ → ミニサムネイル（64×64px）に縮小、左にスライド
2. 新しいポップアップが標準位置に表示
3. 同時表示数は設定可能: 1 / 3 / 5（デフォルト） / 10。上限を超えた古いものは自動ディスミス
4. ミニサムネイルはホバーで展開（200ms）
5. 各ミニサムネイルにインデックスバッジ（1, 2, 3...）

## 6. OCRモード

### 方法1: 専用ホットキー（プライマリ）

**トリガー**: `⌘ ⇧ 4`

1. エリアキャプチャと同じオーバーレイ + 選択
2. 選択中のHUDに `OCR Mode` バッジ（オレンジ）表示
3. 選択後: OCR実行（Vision framework、100-500ms）
4. プレビューポップアップ（テキスト表示版）:

```
┌─────────────────────────────────┐
│  ┌───────────────────────────┐  │
│  │ 抽出テキストプレビュー       │  │
│  │ モノスペース、最大8行、      │  │
│  │ スクロール可能              │  │
│  └───────────────────────────┘  │
│                                 │
│  142 words · 3,847 chars        │
│  ✓ Copied as text               │
│                                 │
│  [Image] [Both] [Save .txt] [×] │
└─────────────────────────────────┘
```

**クリップボード**: 抽出テキスト（プレーンテキスト、UTF-8）

**アクションボタン**:

| ボタン | ショートカット | クリップボード内容 |
|--------|----------|--------------------|
| Image | `I` | 最適化画像に切り替え |
| Both | `B` | リッチクリップボード: 画像+テキスト（NSPasteboard マルチタイプ） |
| Save .txt | `S` | テキストファイルとして保存 |

### 方法2: プレビューポップアップから（セカンダリ）

キャプチャ後のプレビューで `T` を押す / "OCR" クリック → スキャンアニメーション → テキスト表示に切り替わる

### 方法3: ハイブリッド（Auto-OCR設定）

設定でAuto-OCRを有効にすると、全キャプチャでバックグラウンドOCRが実行される。プレビューにOCRテキストのプレビュー行が追加表示。"Both"をデフォルトに設定可能。

## 7. バーストモード

### 動作

明示的な「バーストモード」は存在しない。連続キャプチャが自然にバースト動作になる。

1. `⌘⇧2` → エリア選択 → キャプチャ1がクリップボードに
2. すぐに `⌘⇧2` 再度押下 → キャプチャ2
3. プレビューがスタッキング（上記セクション5参照）

### バッチコピー

**トリガー**: `⌘ ⇧ C`（プレビューポップアップ表示中）

```
┌────────────────────────────────────┐
│  Batch Copy: 4 captures            │
│                                    │
│  [1] [2] [3] [4]  (サムネイル)     │
│                                    │
│  Total: 487 KB · ~5,248 tok        │
│                                    │
│  [Copy All]  [Copy as Grid]        │
│  [Copy OCR]  [Cancel]              │
└────────────────────────────────────┘
```

| オプション | クリップボード内容 |
|-----------|-------------------|
| Copy All Images | 複数画像（Claude/ChatGPT web対応） |
| Copy as Grid | 2×Nグリッドに結合した単一画像 |
| Copy OCR Text | 全キャプチャのOCRテキストを `---` 区切りで連結 |

## 10. アノテーションエディタ

### 10.1 コンセプト

v1 の編集体験は、別ウィンドウのフルエディタではなく **Preview 内の Focus Marker モード** として実装する。

- 目的は「装飾」ではなく、LLM が参照しやすい注目点を最短で埋め込むこと
- サポートする図形は `Pin` と `Rectangle` の 2 種類に限定
- Pin は `1, 2, 3...`、Rectangle は `A, B, C...` で自動識別
- 位置は画像内の相対座標として保持し、補助テキストに `center-right`, `x=12%-38%` のような説明を生成

### 10.2 開き方

- **開き方**: Preview hover の `Annotate` / `E` キー
- **表示方式**: Preview を expand した状態で、画像上にツールバーと prompt preview を重ねる
- **ハッピーパス**: キャプチャ → Expand → Pin/Rect を 1-2 個置く → `Copy for LLM`

### 10.3 ツールセット（v1 実装）

#### 1. Pin

- クリックした位置に赤い Pin を配置
- 識別子は `1, 2, 3...`
- 任意のメモを「次のマーカー用ノート」として入力可能

#### 2. Rectangle

- クリック＆ドラッグで矩形を配置
- 識別子は `A, B, C...`
- 任意のメモを付加可能

#### 3. Undo / Clear

- `Undo`: 直前のマーカーを取り消し
- `Clear`: 全マーカーを削除

### 10.4 Copy for LLM

- マーカーを置くと、**注釈入り画像が自動でクリップボードに再コピー**される
- `Copy for LLM` は image + text の複合 pasteboard を試行する
- paste 先が text を拾わないケースに備えて `Copy Prompt` を常に用意する
- Prompt preview には `Pin 1 at center-right (65% from the left, 48% from the top)` のような補助文を表示する

### 10.5 ワークフロー例

```
1. ユーザーがUIバグを発見
2. ⌘⇧2 でキャプチャ
3. Preview の Annotate を押す
4. Rectangle A で崩れた領域を囲む
5. Pin 1 を問題のボタンに置く
6. Prompt preview を確認
7. Copy for LLM
8. ChatGPT / Claude / Cursor にペースト
```

**所要時間**: 2-4秒（慣れたユーザー）

### キーボードショートカット（エディタ内）

| キー | アクション |
|------|---------|
| `1-5` | ツール選択（Arrow/Rect/Text/Blur/Crop） |
| `⌘Z` / `⌘⇧Z` | Undo / Redo |
| `Enter` | 確定＆コピー＆閉じる |
| `⌘C` | コピー（エディタは開いたまま） |
| `Esc` | 破棄＆閉じる |
| `Delete` | 選択中のアノテーションを削除 |
| `R/G/B/Y` | 色変更（Red/Green/Blue/Yellow） |
| `[` / `]` | ストローク幅 -/+ |
| `Space` ホールド | パン（ズーム時） |
| `⌘0` | ズームリセット |

## 11. 設定

### 11.1 General

- Launch at login
- メニューバーアイコンスタイル（Default / Minimal / Monochrome）
- デフォルトキャプチャアクション
- キャプチャ音 ON/OFF
- プレビュー表示 ON/OFF + 自動ディスミス秒数（3/5/10/Until action）
- プレビュー表示位置（bottom-right / bottom-left / top-right / top-left / near cursor）
- プレビュースタッキング上限（1 / 3 / 5 / 10）

### 11.2 LLM Optimization

- デフォルトプロバイダ
- プロバイダ別プロファイル:
  - Claude: 1568px, WebP, 80%
  - GPT-4o: 2048px, PNG, —
  - Gemini: 768px, JPEG, 85%
- トークン予算（最大トークン数/画像）
- Auto-optimize ON/OFF

### 11.3 Shortcuts

- 各アクションにカスタマイズ可能なホットキー

## 12. エッジケースと対策

### マルチディスプレイ

**v1**: エリア選択は単一ディスプレイに制限。ドラッグがディスプレイ境界を越えた場合、ドラッグ開始ディスプレイにスナップ。クロスディスプレイキャプチャはv2。

### Retina / HiDPI

- 常にネイティブ解像度でキャプチャ（テキストの鮮明さを維持）
- LLM最適化で自動ダウンスケール
- ステータスバーに表示: `Native: 3840×2160 → Claude: 1568×882 (saved ~73% tokens)`
- アノテーションはダウンスケール**後**の解像度でレンダリング（縮小で小さくならない）

### クリップボード管理

**v1**: キャプチャがクリップボードを上書きする際、プレビューに "Replaced clipboard contents" と表示。クリックで前の内容を復元。フルクリップボード履歴はv2。

### パーミッション

- 権限拒否: System Settings への深リンクボタン付きガイド表示
- macOS Sequoiaの定期的な再確認: 「macOSが再確認を求めました。許可してください」と表示
- Accessibility権限なし: グローバルホットキーが動作しない → メニューバークリックでキャプチャ可能 + バナーで誘導

### パフォーマンス目標

| 指標 | 目標 | 最大許容 |
|------|------|---------|
| ホットキー → オーバーレイ表示 | 80ms | 150ms |
| 選択確定 → クリップボード書き込み | 100ms | 200ms |
| オーバーレイ消失アニメーション | 100ms | 150ms |
| プレビューポップアップ表示 | 150ms | 200ms |
| OCR処理 | 200ms | 500ms |
| プロバイダ切替・再最適化 | 30ms | 80ms |
| エディタ起動 | 200ms | 400ms |
| **合計ハッピーパス（ホットキー→クリップボード）** | **選択後180ms** | **選択後350ms** |

### メモリ管理

- メモリ内に最大5キャプチャ保持（リングバッファ）
- 各キャプチャ: ネイティブ画像 + 最適化画像 + サムネイル
- 古いキャプチャは `~/Library/Caches/com.gaze/` にディスクキャッシュ

### アノテーション永続化

アノテーションデータ（ツール種類、位置、プロパティ）はベース画像と別にJSON保存。履歴から再編集可能。

```json
{
  "version": 1,
  "base_image": "capture_2026-03-25_143022.png",
  "annotations": [
    {"type": "arrow", "number": 1, "from": [100, 200], "to": [300, 150], "color": "#FF3B30"},
    {"type": "rect", "number": 2, "bounds": [250, 100, 400, 300], "color": "#FF3B30"},
    {"type": "text", "position": [310, 320], "content": "Expected: blue button"}
  ]
}
```

## 14. 設計判断ログ

| 判断 | 選択 | 却下した案 | 理由 |
|------|------|----------|------|
| プレビューの位置 | 5オプションから設定可能（デフォルト: 右下） | 右下固定 | ユーザーごとにスクリーンレイアウトとワークフローが異なる |
| アクションボタン | サムネイルホバーでリビール | 常時表示 | ポップアップをコンパクトに保つ。Fitts's lawに基づき画像上に近接配置 |
| オートディスミス | 5秒（デフォルト）+ "Until action"オプション追加 | 即座 / なし | パワーユーザーの速度とカジュアルユーザーの確認欲求のバランス。慎重なワークフローのユーザーも支援 |
| キャプチャタイミング | ホットキー押下時 | オーバーレイ表示後 | オーバーレイがキャプチャに映り込まない |
| ウィンドウキャプチャのデフォルト | シャドウなし | シャドウあり | LLM入力にはクリーンな画像が最適 |
| 自動ナンバリング | ツール間で共有シーケンス | ツール別番号 | 「アノテーション3」が一意であるべき |
| デフォルトカラー | 赤 | 青 / 緑 | 一般的なUI背景（白/灰/青）に対して最高コントラスト |
| テキストラベル背景 | 黒半透明ピル | なし / アウトライン | ライト/ダーク両方のスクショで確実に読める |
| ダウンスケール順序 | アノテーション合成後 | 合成前 | アノテーションが出力解像度で鮮明に維持される |
| マルチディスプレイv1 | 単一ディスプレイ限定 | クロスディスプレイ | 複雑度 vs 使用頻度のトレードオフ |
| バーストモード | 明示的モードなし（スタッキングで自然に対応） | 専用モード | 追加の概念を導入しない |
| エディタの開き方 | プレビュークリック | 全キャプチャで自動 | ほとんどのキャプチャはアノテーション不要 |
