# Gaze — 配布・販売戦略

> 作成日: 2026-03-26
> ステータス: 初期策定
> 方針: **B2Bで利益を最大化**するPLG（Product-Led Growth）戦略

---

## Executive Summary

Gazeは「AIに画面を見せる」行為を根本から改善するデスクトップツール。

Claude Desktop やChatGPTアプリにもスクリーンショット機能はあるが、それは「1枚の画像を1つのAIに貼る」だけ。Gazeが解決するのは、その先にある問題:

- **画像で送る必要がないものまで画像で送っている** → OCRテキスト化でトークン95%削減
- **AIエージェントが自分で画面を見られない** → MCP Server内蔵で自律キャプチャ
- **UIフロー全体を伝えるのに1枚ずつ貼っている** → バーストモードで一括送信
- **社内の機密情報がそのままAIに渡っている** → スマートマスキングで自動除去

**戦略の核心**: 個人に買い切り$29でばらまき → チーム内で自然拡散 → 管理機能でB2B課金（PLGモデル）

- **個人Pro**: $29買い切り（2台まで）/ $39/年 / $4.99/月
- **Team**: $12/user/月
- **Enterprise**: Contact Sales

**目標**: 24ヶ月でARR $500K、うちB2B比率60%以上

### 2.2 Gazeの価値提案（再定義: 2026-03-27, CLI-first方針追記）

> 旧: 「LLMプロバイダ別に画像を最適化してトークンコスト削減」
> 新: 「AIに画面を見せる行為を、あらゆる形で最適にする」
> アーキテクチャ: **CLI-first, MCP-also, GUI-too**

**価値提案の再定義の背景**:
- 2026年3月時点で、Claude/GPT-4o/Gemini 3全てが画像の自動リサイズを実装済み
- 画像トークンのコストは1枚$0.002-0.005まで下落。最適化で浮く額は$0.001/枚
- Claude DesktopとChatGPTアプリがスクリーンショット機能を内蔵し、「撮って貼る」を無料で提供

**CLI-first方針の背景（2026-03-27追記）**:
- ベンチマーク実測: CLIはMCPよりトークンコスト10-32倍安く、信頼性100% vs 72%
- Claude Code/Codex CLI/AiderなどターミナルエージェントはBash実行が主力
- Claude Code公式が「CLIツールを使え」と推奨
- `brew install gaze` で即利用可能。MCP設定ファイル編集は不要
- Unixパイプで組み合わせ可能: `gaze capture | gaze ocr | pbcopy`
- 直接競合: Peekaboo（CLI+MCP、macOS汎用自動化）→ Gazeは「LLMインプット特化」で差別化

**新しいコア価値（差別化が成立するもの）**:

| 価値 | CLI コマンド例 | 競合にない理由 |
|------|---------------|--------------|
| **テキスト化** | `gaze ocr` / `gaze ocr --area 100,200,500,400` | トークン95%削減。画像より正確。Peekabooにも基本OCRはあるが、構造化テキスト+トークン数表示は差別化 |
| **エージェント連携** | `gaze capture` をCLIで実行（+ MCP薄ラッパー） | CLI-firstでClaude Code/Codex CLIから即利用可能。MCPはCursor/Windsurf向けに薄ラッパーで提供 |
| **コンテキスト一括送信** | `gaze capture --burst 5 --interval 2` | Peekabooにバースト機能なし。UIフロー全体を一括キャプチャ→送信 |
| **動画キャプチャ** | `gaze record --duration 10 --fps 2` | CLIから動画を撮れるツールは少ない。動画→フレーム抽出→OCRのパイプライン |
| **機密情報の保護** | `gaze capture --mask-pii` | スクショ特化のPIIマスキング。送信前に自動除去 |
| **パイプライン** | `gaze capture \| gaze ocr \| gaze mask` | Unix哲学に基づく組み合わせ可能性。Peekabooにはない |

**CLIコマンド体系（想定）**:
```bash
gaze capture                              # スクショ→stdout (PNG)
gaze capture --area 100,200,500,400       # 範囲指定
gaze capture --window "Chrome"            # ウィンドウ指定
gaze capture --burst 5 --interval 2       # 5枚連続、2秒間隔
gaze record --duration 10 --fps 2         # 10秒動画
gaze ocr                                  # 画面全体をOCRテキスト化
gaze ocr --area 100,200,500,400           # 範囲指定OCR
gaze capture | gaze ocr               # パイプでキャプチャ→OCR
gaze capture | gaze mask              # パイプでキャプチャ→PIIマスク
gaze watch --interval 5 --ocr --diff      # 画面監視（差分検知）
```

**3つのインターフェース**:
1. **CLI**（エージェント・自動化向け）— 最優先で実装。`brew install gaze`
2. **MCP**（IDE統合向け）— CLIの薄ラッパー。Cursor/Windsurf対応
3. **GUI**（人間向け）— Tauriアプリ。ショートカットキーで起動。従来のスクショツール体験

**Peekabooとの差別化**:

| 軸 | Peekaboo | Gaze |
|----|----------|---------|
| 方向性 | 汎用macOS自動化（click, type, scroll） | LLMインプット特化 |
| OCR | 基本的なテキスト抽出 | 構造化テキスト + トークン数表示 |
| 動画 | なし | 動画キャプチャ + フレーム抽出 |
| バースト | なし | 連続キャプチャ→一括送信 |
| パイプライン | なし | `capture | ocr | mask` のUnixパイプ |
| Windows | macOS専用 | クロスプラットフォーム |
| GUI | なし | Tauriアプリ（人間用UI） |

**付随機能（あれば嬉しいがコア訴求にはしない）**:
- LLMプロバイダ別画像リサイズ（各社が自動対応済みのため格下げ）
- 1キーストロークのキャプチャスピード（Claude Desktop等が無料提供のためテーブルステークス化）
- 軽量15MB / Rust製（購入動機にはならない）

**セグメント別の訴求メッセージ**:

| セグメント | 刺さるメッセージ |
|-----------|----------------|
| **個人開発者** | 「スクショをテキストで送れる。画像より速い、安い、正確」 |
| **チーム** | 「バグ報告のUIフロー全体を1アクションでAIに見せる。チーム全員が同じ品質で」 |
| **エンタープライズ** | 「社内画面のスクショからAPIキーや個人情報を自動マスク。AIに安全に渡せる」 |
| **AIエージェント開発者** | 「MCPで接続すれば、エージェントが自分で画面を見に行く」 |

### 2.3 ローカル処理の戦略的優位性

Gazeの**ローカル処理アーキテクチャ**はB2Bにおいて最大の武器：

- DPAの範囲が大幅に限定される（画像データは処理対象外）
- セキュリティチェックシートの多くが「該当なし」に
- GDPR/個人情報保護法の越境移転問題を回避
- **「画像データはデバイスから一切外部に送信されません」**が営業上の最強メッセージ

## 3. プライシング戦略

### 3.1 モデル: 買い切り + サブスク ハイブリッド（個人）+ Per-seat サブスク（B2B）

個人向けは**買い切りを主軸に据え、サブスクも選択可能**。B2Bはper-seatサブスク。
買い切りが最もお得になる設計で、ユーザーに買い切りを自然に選ばせる（LTV予測が立てやすい）。

#### 個人向け（買い切り + サブスク選択制）

| プラン | 価格 | 内容 |
|--------|------|------|
| **Free** | $0 | 月50回キャプチャ、基本機能のみ |
| **Pro（買い切り）** | **$29** | 全Pro機能 + **1年間アップデート** + **2台まで** |
| **Pro（アップデート更新）** | **$15/年**（任意） | 2年目以降のアップデート継続。更新しなくても最終版は永久利用可 |
| **Pro（年額サブスク）** | **$39/年** ($3.25/月相当) | 常に最新版 + 全Pro機能 |
| **Pro（月額サブスク）** | **$4.99/月** | お試し・短期利用向け。常に最新版 |

**価格バランスの設計思想**:
- 買い切り $29 が最もお得（年額サブスク $39/年の75%、月額の48%）
- 年額サブスク $39/年 は買い切りの約1.3倍 — 業界標準の範囲
- 月額 $4.99 は年額比 +53% — 短期利用・お試しのプレミアム（業界標準 20-50%）
- **参考**: TablePlus $89買い切り(1台)、CleanShot X $29買い切り(1台)、Sublime Text $99買い切り(3台3年)

**2台ライセンスの差別化**:
- CleanShot X (1台)、TablePlus (1台) に対して明確な優位性
- 将来のMac + Windows対応時に「1ライセンスで両方使える」訴求が可能

#### B2B向け（Per-seat サブスクのみ）

| プラン | 価格 | 対象 | 年払い |
|--------|------|------|--------|
| **Team** | $12/user/月 | チーム（3人〜） | $120/user/年（17%割引） |
| **Enterprise** | Contact Sales | 50人以上の組織 | 年間契約必須 |

B2Bで買い切りを提供しない理由: チーム管理機能はサーバーサイド維持が必要、ユーザー増減にサブスクが適合。

### 3.3 価格設定の根拠

**Pro買い切り $29**:
- CleanShot X ($29) と同価格帯で「納得感のある価格」
- 2台まで対応でCleanShot X (1台) より明確にお得
- 1年間アップデート + 更新$15/年（CleanShot X: $19/年）も低め設定
- 日本円で約¥4,500 — 開発者が即決できるライン

**Pro月額 $4.99/月**:
- 個人が自腹で払える上限（日本: 約¥750/月）
- 買い切りに誘導する価格設計だが、お試し需要はカバー
- 企業内浸透のフックとして機能（個人Pro → チーム提案のトリガー）

**Team $12/user/月**:
- Raycast Teams ($12/user/月)、Loom Business ($15/user/月) と同等水準
- 日本企業の「¥1,000-3,000/月/人」の部門裁量枠に収まる（約¥1,800）
- 個人Pro買い切り ($29) から大きくステップアップ — 管理機能・共有機能の明確な価値差

**Enterprise（$25+/user/月相当）**:
- Team の約2倍（業界標準の2-4倍レンジの下限）
- SSO/SCIM は大企業の導入必須条件（いわゆる「SSO Tax」）
- 最低年間契約額: $10,000（ARR floor）
- ボリュームディスカウント: 100席以上で15-20%

### 3.4 ボリュームディスカウント（Team）

| 席数 | 割引率 | 実質単価 |
|------|--------|---------|
| 3-10 | 0% | $12/user/月 |
| 11-25 | 10% | $10.80/user/月 |
| 26-50 | 15% | $10.20/user/月 |
| 51+ | Contact Sales | 交渉制 |

### 3.5 PLGファネル設計

```
[Stage 1] 個人開発者がFreeで使い始める
    ↓  月50回制限に到達（通常1-2週間）
[Stage 2] Pro 買い切り $29 で購入（最もお得な選択肢として誘導）
    ↓  チーム内で「これ便利」と共有（1-3ヶ月）
[Stage 3] 3-5人が個別にPro購入
    ↓  管理の必要性 + 共有プリセットの需要が発生
[Stage 4] Team $12/user/月 にアップグレード（部門長決裁）
    ↓  他部門に波及（3-6ヶ月）
[Stage 5] IT部門が「全社統一」を検討 → SSO要件 → Enterprise
```

**転換トリガー設計**:
- Free → Pro: **キャプチャ回数制限**（月50回は業務利用で1-2週間で到達）
- Pro → Team: **「チームに共有」ボタンの露出** + **個別Proが3人以上になった時のアップグレード提案**
- Team → Enterprise: **SSO/SCIMの要求**（情シスの導入条件）

## 4. 配布チャネル戦略

### 4.1 配布チャネル

| チャネル | 優先度 | フェーズ | 理由 |
|---------|--------|---------|------|
| **直販（自社サイト）** | 最高 | Day 1 | 利益率最大、サンドボックス制約なし |
| **Homebrew Cask** | 高 | Day 1 | 開発者の標準インストール方法 |
| **Winget** | 高 | Windows対応時 | Windows開発者向け |
| **Setapp** | 中 | Month 6+ | 安定収入 + Mac市場のディスカバリー |
| **Mac App Store** | 低 | 検討のみ | サンドボックスとスクリーンキャプチャの相性問題 |

### 4.2 決済・ライセンス基盤

**Phase 1（MVP〜Month 6）**: LemonSqueezy
- MoR（Merchant of Record）として税務処理代行
- ライセンスキー管理が内蔵
- 手数料: 5% + $0.50/トランザクション
- セットアップが最速（1-2日で稼働可能）

**Phase 2（Month 7〜）**: Paddle + Keygen
- Paddle: 手数料が透明（5% + $0.50）、B2B決済に強い
- Keygen: Tauri公式連携あり、SOC2 Type II取得済
  - チームライセンス、フローティングライセンス対応
  - $299/月〜（開発中は無料）

**日本市場向け追加対応**:
- 円建て請求（Paddle/LemonSqueezyで対応可）
- 請求書払い（銀行振込）: Enterprise契約時に手動対応 → 将来freee/MF連携
- インボイス制度対応（適格請求書発行事業者登録）

### 4.3 技術的な配布要件

| 要件 | macOS | Windows |
|------|-------|---------|
| **コード署名** | Apple Developer Program ($99/年) + Notarization | OV証明書 ($200-400/年) |
| **インストーラー** | `.dmg`（標準）、`.pkg`（MDM向け） | `.msi`（Tauri標準生成） |
| **自動アップデート** | Tauri Updater（内蔵） | Tauri Updater（内蔵） |
| **MDM対応** | `.pkg` + Jamf/Mosyle/Kandji | `.msi` + Intune/SCCM |

## 6. 営業モデル

### 6.1 セグメント別営業手法

| セグメント | 手法 | 担当 |
|-----------|------|------|
| 個人（Free/Pro） | **完全セルフサーブ** | プロダクト（自動） |
| チーム（Team） | **セルフサーブ + チャットサポート** | 創業者 or CS |
| エンタープライズ | **セールスアシスト** | 創業者（初期）→ AE |

### 6.2 営業の原則

1. **初期はインバウンド100%** — 開発者ツールでアウトバウンド営業は嫌悪される
2. **コンテンツで集客** — 技術ブログ + コミュニティが最も効果的
3. **チャンピオン支援** — 社内で推してくれるユーザーに稟議用資料を提供
4. **アウトバウンドはARR $500K超から** — 特定業界への垂直展開時
